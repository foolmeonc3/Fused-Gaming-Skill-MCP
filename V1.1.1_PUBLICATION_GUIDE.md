## 📦 Step 3: Publish to npm

### Pre-Publication Validation

```bash
# Ensure you're on main branch
git checkout main
git pull origin main

# Run full validation suite
npm run build
npm run typecheck
npm run lint
npm test

# Verify version in package.json and VERSION.json
grep '"version"' package.json
cat VERSION.json | grep '"version"'

# Should both show: "version": "1.1.1"
```

### Login to npm

```bash
npm login
# Enter your npm username, password, and email
# If using 2FA, you'll get a prompt for your one-time password
```

### Publish Packages

```bash
# Option A: Publish all workspace packages (RECOMMENDED)
npm run publish:packages

# Option B: Publish individual packages
npm publish --workspace=packages/core
npm publish --workspace=packages/cli
npm publish --workspace=packages/skills/*
```

### Monitor Publication

```bash
# Watch for output like:
# npm notice 
# npm notice 📦  @h4shed/mcp-cli@1.1.1
# npm notice === Tarball Contents ===

# Verify packages are published (wait 2-5 minutes for mirror sync)
npm view @h4shed/mcp-cli@1.1.1
npm view @h4shed/mcp-core@1.1.1
npm view @h4shed/skill-algorithmic-art@1.1.1
```

### Troubleshooting

**Error: "You do not have permission to publish"**
- Ensure you're logged in: `npm whoami`
- Verify you have publish access to @h4shed scope
- Contact @h4shed maintainers

**Error: "version already exists"**
- v1.1.1 may already be published
- Check with: `npm view @h4shed/mcp-cli versions`
- If published, verify publication is correct and skip

**Error: "version is behind the latest"**
- Ensure your local version is updated
- Run: `git pull origin main && npm run publish:prepare`

## 🆘 If Something Goes Wrong

### Tag Push Failed
```bash
# Verify local tag exists
git tag -l v1.1.1

# Try pushing with different protocol
git remote -v
# If using http, could try ssh or https with token
```

### npm Publish Failed
```bash
# Check npm login status
npm whoami

# Re-login
npm logout
npm login

# Try publishing single package to test
npm publish --workspace=packages/core --dry-run
```

### GitHub Release Not Showing
```bash
# Verify tag was pushed
git ls-remote origin refs/tags/v1.1.1

# Refresh GitHub page or create release manually from UI
```
