## 🔄 Version Numbering Convention

### Semantic Versioning (SemVer)
- **Major (X.0.0):** Breaking changes, major feature overhaul
- **Minor (1.X.0):** New features, backwards-compatible
- **Patch (1.0.X):** Bug fixes, performance improvements

### Release Frequency
- **Major:** Quarterly (or as needed for significant milestones)
- **Minor:** Monthly (planned new skills or features)
- **Patch:** As needed (security fixes, hotfixes)

---

## 🔧 Publishing Workflow

1. **Branch:** Develop on feature/skill branches
2. **Review:** PR with changelog and version bump
3. **CI/CD:** Run tests, lint, typecheck, build
4. **Publish:** Use `npm run publish:packages` on main branch
5. **Tag:** Create GitHub release tags (v1.2.0, etc.)
6. **Announce:** Update docs, CHANGELOG, README
